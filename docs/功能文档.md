# 功能文档 · AI English Speaking Coach

本文档面向开发者与评审，说明各功能的设计与技术实现。

---

## 功能总览

| 模块 | 功能 | 技术实现 |
|---|---|---|
| 场景对话 | 多场景 + 难度 + 角色扮演 | DeepSeek + 场景化 system prompt |
| 实时语音 | 录音 → 转写 → 回复 → 朗读 | MediaRecorder + Azure STT + DeepSeek + WebSocket + speechSynthesis |
| 发音评测 | Accuracy / Fluency / Expression 三维度 | Azure 发音评测 + WPM 计算 + DeepSeek |
| 实时纠错 | 语法/表达纠错 | DeepSeek 结构化输出 |
| 选词工具 | 查义 / 翻译 / 朗读 | Dictionary API + /api/translate + speechSynthesis |
| 跨会话记忆 | 记住薄弱项 | 会话数据持久化 + 开场回顾 |
| 课后报告 | 多维度量化总结 | DeepSeek 分析 + 可视化 |
| 用户系统 | 登录 / 额度 / Pro | Clerk + SQLite 计数 + 爱发电 Webhook |

---

## 1. 多场景对话

- **日常场景**：interview / restaurant / meeting / hospital / phone_call / customer_service
- **程序员面试**：sde_behavioral / sde_project / sde_thinking
- **难度**：easy / medium / hard，影响 AI 用词与追问深度。
- **实现**：后端根据 scene + difficulty 构造 system prompt，DeepSeek 扮演对应角色（面试官 / 服务员等），保持对话自然与角色一致性。

---

## 2. 实时语音对话（端到端语音闭环）

**数据流：**

```
MediaRecorder 录音 (webm/opus)
   → 前端用 Web Audio API 解码并转成 16kHz 16-bit 单声道 WAV
   → POST /api/pronunciation-assess（含音频 + duration_ms）
   → Azure STT 转写得到文本
   → 文本经 WebSocket 发送给后端
   → DeepSeek 生成角色回复 + 纠错结果
   → 前端 speechSynthesis 朗读 AI 回复
```

**关键技术点：**
- 浏览器 MediaRecorder 录出的是 webm/opus，Azure REST 对其兼容性差，因此前端用 `AudioContext.decodeAudioData` 解码后自行写 WAV 头转成 PCM WAV 再上传。
- 采用 WebSocket 维持对话上下文，降低交互延迟。
- 支持 Auto / Manual 两种发送模式。

---

## 3. 发音评测（三维度）

| 维度 | 来源 | 实现 |
|---|---|---|
| **Accuracy** | Azure 发音评测 | 两步法：先用 Azure STT 拿到转写文本，再以该文本作为 ReferenceText 调用发音评测（Pronunciation-Assessment header），逐词得到 AccuracyScore，汇总为整体准确度。同时返回逐词得分用于「待改进单词」。 |
| **Fluency** | WPM 计算 | 后端用 `词数 / (duration_ms/60000)` 得到语速，映射到 0–100：110–170 WPM 视为自然区间，过慢/过快递减。无需额外 API 调用。 |
| **Expression** | DeepSeek | 把转写文本交给 DeepSeek，按「**地道度而非高级词汇**」打分，并产出「把 X 改成 Y」式中文具体建议。 |

> Azure REST API 仅稳定返回 AccuracyScore（FluencyScore/PronScore 需原生 SDK），故 Fluency 由 WPM 自算、Expression 由 LLM 评判，三者互补且各有明确含义。

性能优化：Azure 评分与 DeepSeek 表达评分通过 `asyncio.gather` **并行执行**，减少总等待时间。

---

## 4. 实时纠错

每轮对话后，DeepSeek 以结构化格式返回：
- `has_error`、`original`（原句）、`corrected`（改后）、`explanation`（解释）、`error_type`（错误类型）。
前端在右侧面板以「原句 → 改后 + 解释」对照展示。

---

## 5. 选词查义 / 选句翻译 / 朗读（WordTooltip）

- **选中单个单词**：调用 Free Dictionary API 获取音标 + 释义；查不到则回退到 `/api/translate` 给出中文。
- **选中短语/句子**：调用后端 `/api/translate`（DeepSeek 翻译）显示中文。
- **朗读**：均提供 🔊 按钮，使用浏览器 `speechSynthesis`（优先 Google US English 音色）。
- **实现要点**：组件用原生 `addEventListener('mouseup')` 监听选区；popup 内点击通过 `data-tooltip-popup` 标记跳过查词逻辑（避免 native 事件绕过 React `stopPropagation` 导致 tooltip 被清除）；朗读用 `setTimeout` 规避 Chrome `cancel()+speak()` 同 tick 静默失败的问题。

---

## 6. 跨会话记忆

- 会话内容与薄弱项持久化到 SQLite。
- 同场景再次练习时，后端在开场 greeting 中注入上次薄弱项，AI 主动回顾并针对性训练（前端显示 🧠 记忆横幅）。

---

## 7. 课后报告与历史

- **报告**：结束会话后由 DeepSeek 分析整段对话，产出整体评分、语法错误、表达清晰度、结构等多维度结果。
- **历史**：记录每次练习分数，前端可视化展示提升趋势。

---

## 8. 用户系统与商业化

- **认证**：Clerk（前端 publishable key + 后端 secret key 校验 JWT）。
- **额度**：SQLite 按 `user_id + 日期` 计数；未登录用户按 IP 计数，免费上限更低；超限返回 HTTP 429。
- **Pro 会员**：通过爱发电支付，Webhook（`/api/afdian`）回调后写入 `pro_users` 表，Pro 用户不限次。

---

## 9. 主要 API 端点

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/health` | 健康检查 |
| POST | `/api/session` | 创建练习会话 |
| WS | `/ws/{session_id}` | 实时对话 |
| POST | `/api/pronunciation-assess` | 发音评测（三维度） |
| GET | `/api/user/status` | 用户额度状态 |
| GET | `/api/translate` | 文本翻译（选句翻译） |
| GET | `/api/session/{id}` | 获取会话 |
| POST | `/api/session/{id}/end` | 结束会话 |
| POST | `/api/afdian` | 爱发电支付回调 |

---

## 10. 部署

- **前端**：Vercel（Next.js）。
- **后端**：Railway（FastAPI，Nixpacks 构建，`uvicorn main:app`）。
- **密钥管理**：全部通过平台环境变量注入，仓库内不含任何真实密钥。
